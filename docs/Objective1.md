# WaterWise System Development Plan (Objective 1)


| Week | Day | Task Description | Sub-Tasks (Breakdown) | Deliverable(s) | Test Suite / PR Acceptance Criteria |
|------|-----|------------------|------------------------|----------------|-------------------------------------|
| **Week 1** | **Day 1** | Create Login Module | - Create Login Forms<br>- Create Header<br>- Create Roles Table UI | - Login Forms<br>- Header<br>- Roles Table | - It should allow users to login using their username and password successfully.<br>- It should navigate users to the admin dashboard when valid credentials are provided.<br>- It should display all available system roles and assigned permissions correctly, allowing administrators to create, update, and manage role access configurations successfully. |
| **Week 2** | **Day 1** | Create Dashboard Module | - Create Admin KPI Cards for Overall Consumption<br>- Create Admin KPI Cards for Monthly Consumption<br>- Create Admin KPI Cards for Yearly Consumption<br>- Create Admin KPI Cards for Per Purok Consumption | - Admin Dashboard Page<br>- Admin KPI Cards | - It should render one KPI card for each dashboard data item.<br>- It should display the title, metric value, and subtitle for Overall, Monthly, Yearly, and Per Purok Consumption KPI cards.<br>- It should render default values when KPI information is missing.<br>- It should display KPI cards in the correct order. |
| **Week 2** | **Day 2** | Create Consumer Management Module | - Create Consumer Form UI<br>- Create Payment Form UI<br>- Create Consumer List UI | - Consumer Form<br>- Consumer List Table<br>- Payment Form | - It should accept and validate consumer information including Account Name, Full Name, Contact Number, Purok, and Email Address.<br>- It should retrieve and display all registered consumer records from the database.<br>- It should record consumer payment transactions successfully, calculate total amount due accurately, generate payment records, and update consumer payment status after submission. |
| **Week 3** | **Day 1** | Create Event Tagging & Announcement Management Module | - Create Event Form<br>- Store Event Records<br>- Create Announcement Form UI<br>- Broadcast Announcements<br>- Create Event Backend<br>- Create Announcement Backend | - Event Form<br>- Event Records Table<br>- Announcement Form<br>- Announcement Page<br>- Event Backend<br>- Announcement Backend | - It should allow authorized users to create event records with required details such as event title, description, date, time, location, and event tags.<br>- It should display event details including title, schedule, location, status, and assigned tags.<br>- It should allow authorized users to create announcements containing title, content, publication date, and related event information.<br>- It should display announcement details correctly. |
| **Week 3** | **Day 2** | Create Authentication Backend | - Create Authentication Service<br>- Implement Login/Logout Logic<br>- Protect Routes<br>- Manage User Sessions | - Authentication Service (`AuthService.js`)<br>- Protected Route Middleware<br>- Session Management Logic<br>- Login & Logout Backend Integration | - It should authenticate users successfully using valid email and password credentials.<br>- It should establish a user session upon login.<br>- It should return appropriate error messages for invalid authentication attempts.<br>- It should restore active sessions after page refresh.<br>- It should allow users to logout successfully and clear active sessions.<br>- It should prevent unauthorized users from accessing restricted modules.<br>- Unit tests should cover authentication services, session handling, protected routes, and mocked authentication responses. |
| **Week 4** | **Day 1** | Create Consumer & Meter Reading Backend | - Create Reading Database<br>- Store Meter Reading Records<br>- Create Consumer Database | - Consumer Database<br>- Reading Database | - It should store, retrieve, update, and delete consumer records successfully.<br>- It should save meter reading records accurately.<br>- It should associate readings with the correct consumer account.<br>- It should retrieve historical reading data without errors. |
| **Week 4** | **Day 2** | Create Billing & Payment Backend | - Create Billing Computation Logic<br>- Generate Billing Records<br>- Record Payment Transactions | - Billing Computation Service<br>- Billing Records<br>- Payment Transaction Service | - It should calculate billing amounts accurately based on water consumption, rate schedules, penalties, and applicable charges.<br>- It should generate, store, and retrieve billing records successfully.<br>- It should record payment transactions successfully and update billing status (Paid, Partially Paid, Unpaid). |
| **Week 5** | **Day 1** | Create Role-Based Access Control Module | - Create Roles Table<br>- Create Consumer Form UI<br>- Create Reading Form UI | - Roles Table<br>- Consumer Form<br>- Reading Form | - It should render Consumer Management modules only for users with required permissions.<br>- Authorized users should create and update consumer records successfully.<br>- Unauthorized users should be restricted from accessing consumer features.<br>- It should render Meter Reading modules only for users with Meter Reading permissions.<br>- Authorized users should submit, edit, and save meter reading records successfully. |
| **Week 5** | **Day 2** | Create Report Generation Module | - Create Report Templates<br>- Generate PDF Reports | - PDF Report Generator<br>- Generated Reports | - It should support configurable report sections, headers, footers, and data fields.<br>- It should allow administrators to view, download, and print generated PDF reports. |
| **Week 6** | **Day 1** | Overall System Testing and Validation | - Test All Modules<br>- Validate System Objectives | - Final Tested System<br>- Test Report | - It should complete the end-to-end workflow successfully including authentication, consumer management, meter reading entry, billing computation, payment processing, and report generation.<br>- It should provide evidence that all major modules have been tested including Authentication, Consumer Management, Meter Reading, Billing, Payments, and Reporting. |
